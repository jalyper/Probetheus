## Critical Gap: Save/Load Persistence

### What's Missing

**probe.shellId not serialized**

Location: `src/SaveManager.js` lines 99-116 (probe serialization)

The `equipShellOnProbe()` method correctly sets `probe.shellId`, but `SaveManager.createSaveData()` does not include `shellId` in the probe data it serializes.

**refreshProbeCosmetic() not called after restore**

Location: `src/SaveManager.js` lines 487-530 (probe restoration)

The `refreshProbeCosmetic()` helper was created specifically for save/load scenarios, but it is never actually called after probes are restored from a save file.

### Impact

When a player:
1. Equips a custom shell on a specific probe
2. Saves their game
3. Loads that save

The probe will revert to the default shell, losing the per-probe customization. This breaks player expectations for persistence.

### Fix Required

```javascript
// In SaveManager.createSaveData(), probe serialization:
shellId: probe.shellId || 'default',

// In SaveManager.restoreFromData(), after probe restoration:
if (window.game?.shellSystem) {
    this.gameState.entities.probes.forEach(probe => {
        window.game.shellSystem.refreshProbeCosmetic(probe);
    });
}
```
