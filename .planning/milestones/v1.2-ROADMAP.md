# Milestone v1.2: Shell Bonuses & Effects

**Status:** SHIPPED 2026-02-02
**Phases:** 1-4 (continuing from v1.1)
**Total Plans:** 9

## Overview

Make shell bonuses functional in gameplay -- equipping a shell with a bonus actually affects the entity's behavior, and players can see what bonuses their shells provide.

## Phases

**Goal:** Implement shell visual system for probes -- equipping shells changes probe body color, trail color, and glow.
**Depends on:** None (ShellSystem already exists)
**Plans:** 1 plan

### Phase 3: Bonus Gameplay

**Goal:** Equipping a shell with a bonus changes how that entity performs in-game.
**Depends on:** None (ShellSystem bonus infrastructure already exists)
**Plans:** 4 plans

Plans:
- [x] 03-01: Normalize shell bonus values and add per-entity bonus resolver

- [x] 03-03: Wire hub and mining station bonuses (researchSpeed, miningEfficiency, probethiumRate, shuttleSpeed)

## Milestone Summary

**Key Decisions:**

| Decision | Rationale |
|----------|-----------|
| Per-entity bonuses (not global stacking) | Intuitive -- each entity's shell affects only itself |
| 2-phase roadmap (gameplay then UI) | All bonus wiring is same pattern; UI depends on working bonuses |
| Remove researchSpeed from probe shells | Probes don't research; only hub-appropriate |
| Remove probethiumRate from probe shells | Probes don't mine; only station-appropriate |
| explorationRewards before artifactDiscovery | Multiplicative stacking: general boost then specific |
| baseMaxDamage tracking on probes | Prevents compounding when swapping shells |
| Research cost reduction: Math.max(1, Math.ceil(...)) | Conservative rounding; minimum 1 prevents free research |
| 300ms tooltip delay | Standard UX pattern prevents accidental tooltips |

**Issues Resolved:**
- baseMaxDamage not being persisted in SaveManager (fixed in 04-03)
- Shell bonus values inconsistent across categories (normalized in 03-01)
- Mismatched bonuses on entity types (cleaned in 03-01)
- Starting probes missing damage/baseMaxDamage properties (fixed in 03-02)

**Issues Deferred:**
- Hub and mining station shells not visually applied (deferred from v1.1)
- 3/11 tooltip tests have timing issues (test environment specific, not product bugs)

**Technical Debt Incurred:**
