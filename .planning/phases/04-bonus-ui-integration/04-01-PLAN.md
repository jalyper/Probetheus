  <action>
Add these methods to the UIManager class (before the closing brace of the class):

1. `createBonusTooltip()` — Called once during UIManager constructor. Creates a singleton div#bonusTooltip appended to document.body:
   - `position: fixed; display: none; pointer-events: none; z-index: 10001`
   - Dark background: `rgba(20, 20, 40, 0.95)` with subtle cyan border: `1px solid rgba(0, 255, 255, 0.3)`
   - `border-radius: 4px; padding: 8px 12px; color: #ccc; font-size: 11px; line-height: 1.6`
   - CSS `::after` pseudo-element for downward arrow pointer (use a child div instead since this is dynamic): Add a small div with `width: 0; height: 0; border-left: 6px solid transparent; border-right: 6px solid transparent; border-top: 6px solid rgba(20, 20, 40, 0.95)` positioned at bottom center of tooltip via `position: absolute; bottom: -6px; left: 50%; transform: translateX(-50%)`
   - Store reference as `this.bonusTooltip` and `this.tooltipArrow`

2. `buildTooltipContent(shell)` — Returns HTML string or null:
   - If shell has no bonuses or bonuses is empty object, return null
   - For each bonus in shell.bonuses, get info via `window.game.shellSystem.getBonusTypeInfo(bonusType)`
   - Build line: `${info.icon} ${info.label}: <span style="color: #0f0">+${value}${info.unit}</span>`
   - Join lines with `<br>`
   - Return the HTML string

3. `positionTooltip(target)` — Positions tooltip above the target element:
   - Get target rect via `getBoundingClientRect()`
   - Show tooltip briefly off-screen (`left: -9999px`) to measure its dimensions
   - Center horizontally above target: `x = rect.left + (rect.width / 2) - (tooltipWidth / 2)`
   - Position above with 8px gap: `y = rect.top - tooltipHeight - 8`
   - Bounds checking: `x = Math.max(10, Math.min(x, window.innerWidth - tooltipWidth - 10))`
   - If y < 10, flip below: `y = rect.bottom + 8` and move arrow to top of tooltip (flip arrow border directions)
   - Set tooltip `left` and `top` in pixels

4. `attachTooltipHandlers(element, shell)` — Attaches mouseenter/mouseleave:
   - On mouseenter: Set a 300ms timeout. After delay, call `buildTooltipContent(shell)`. If content is null, do nothing. Otherwise set tooltip innerHTML to content, call `positionTooltip(element)`, set display to 'block'. Store timeout ID on element as `element._tooltipTimeout`.
   - On mouseleave: Clear timeout via `element._tooltipTimeout`. Set tooltip display to 'none'.

5. Call `this.createBonusTooltip()` at end of UIManager constructor.

6. In `showShellModal(probe)`: After the existing mouseenter/mouseleave scale handlers (line ~1648), add tooltip handlers. For each shell-option, find the matching shell object from ownedShells using `data-shell-id`, then call `this.attachTooltipHandlers(option, matchingShell)`. Also hide tooltip when modal is closed (in close button handler and background click handler, set `this.bonusTooltip.style.display = 'none'`).

7. In `updateShellSection(probe)`: After setting previewBox innerHTML and nameEl (line ~1488), attach tooltip handler to the previewBox: call `this.attachTooltipHandlers(previewBox, shell)` where shell is the result of `shellSystem.getProbeShell(probe)`.
