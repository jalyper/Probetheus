**Test 1 — TEST-07: Equip shell with bonus, verify effect, swap shell, verify change**
- Start game

- Equip first shell on probe: `shellSystem.equipShellOnProbe(probe, shellId1)`

- Swap to second shell: `shellSystem.equipShellOnProbe(probe, shellId2)`
- Read bonus value again -- should be different (or 0 if swapped to default)
- Return both values for assertion
- Assert values are different

**Test 2 — TEST-07 variant: Swap to default removes bonus**
- Equip a shell with bonuses on probe
- Verify bonus > 0
- Equip 'default' shell
- Verify bonus === 0

**Test 3 — TEST-08: Save/load preserves shell bonuses and they remain functional**
- Start game
- Find a probe and a shell with bonuses (e.g., signalRange)
- Add shell to ownedShells, equip on probe

- Save game: `window.game.saveManager.saveGame()`
- Reload page
- Wait for page load
- Start a new game, then load saved game: `window.game.saveManager.loadGame()`
- Wait for load to complete
- Find the probe again
- Verify probe.shellId matches saved value
- Verify `getEntityBonus('probes', probe, bonusType)` returns same value as before save
- Return all values for assertion

**Test 4 — TEST-08 variant: Hub and station shell bonuses survive save/load**

- Save game
- Reload page, load game
- Verify equippedShells.hubs and .miningStations match
- Verify getEntityBonus returns correct values for both

Follow the save/load test patterns from `tests/save-system.spec.js` for page reload and timing.
