**Test Categories:**

**1. Sector-Specific Spawning (TEST-01 - Requirements SIG-01 through SIG-04):**
- Test: "Ore Vein signals spawn in Resource-Rich sectors"
  - Force spawn Resource-Rich sector via game state manipulation
  - Generate 50-100 signals in that sector
  - Count signals with type='ore_vein'
  - Assert: 10-40% of signals are Ore Vein (15-30% ±10% for statistical variance)
  - Assert: Ore Vein signals do NOT appear in other sector types

- Test: "Data Cache signals spawn in Data Haven sectors"
  - Same pattern as above for Data Haven + data_cache

- Test: "Relic signals spawn in Ancient sectors"
  - Same pattern as above for Ancient + relic

- Test: "Exotic Crystal signals spawn in Asteroid Field sectors"
  - Same pattern as above for Asteroid Field + exotic_crystal

- Test: "Standard sectors spawn NO exclusive signals"
  - Force spawn Standard sector
  - Generate 100 signals
  - Assert: No signals with type='ore_vein' | 'data_cache' | 'relic' | 'exotic_crystal'
  - Assert: All signals are 'mineral' | 'data' | 'artifact' | 'mixed'

**2. Equipment Auto-Collection (TEST-01 continued):**
- Test: "Mineral collector auto-collects Ore Vein signals"
  - Equip probe with mineral_collector
  - Spawn ore_vein signal near probe in Resource-Rich sector
  - Wait for auto-collection cycle (1 second)
  - Assert: Probe cargo.minerals increased
  - Assert: ore_vein signal removed from gameState.entities.signals

- Test: "Data collector auto-collects Data Cache signals"
  - Same pattern for data_collector + data_cache

- Test: "Artifact collector auto-collects Relic signals"
  - Same pattern for artifact_collector + relic

- Test: "Universal collector auto-collects all exclusive signals"
  - Equip probe with universal_collector
  - Spawn one of each exclusive type near probe
  - Wait for auto-collection cycles
  - Assert: All four exclusive signal types collected

**3. Shell Bonus Integration (TEST-02 - Requirement SIG-07):**
- Test: "dataSignalDiscovery bonus increases exclusive signal spawn rate"
  - Equip probe with shell having dataSignalDiscovery: 20%
  - Deploy probe to Resource-Rich sector
  - Generate 100 signals with bonus active
  - Generate 100 signals without bonus
  - Assert: Bonus group has 15-25% more ore_vein signals (statistical test with tolerance)

- Test: "signalRange bonus affects exclusive signal auto-collection"
  - Equip probe with shell having signalRange: 25%
  - Spawn ore_vein signal at distance 85 units from probe (beyond base 80 range, within bonus range ~100)
  - Assert: Signal collected with bonus
  - Remove shell, spawn new signal at same distance
  - Assert: Signal NOT collected without bonus

- Test: "rareSignalChance bonus increases exclusive signal rarity"
  - Equip probe with shell having rareSignalChance: 20%
  - Generate 100 exclusive signals (any type)
  - Count rare/epic/legendary signals
  - Assert: Higher proportion of rare+ signals compared to baseline (no shell)

**4. Save/Load Persistence (TEST-04):**
- Test: "Exclusive signals persist through save/load"
  - Spawn exclusive signals in multiple sectors
  - Record signal types and positions
  - Save game via saveManager.saveGame()
  - Reload page and load save via saveManager.loadGame()
  - Assert: Same exclusive signal types exist at same positions

**5. Regression Protection:**
- Test: "Standard signal types continue working unchanged"
  - Generate signals in Standard sector
  - Assert: mineral, data, artifact, mixed signals still spawn
  - Assert: Existing equipment still collects standard signals
