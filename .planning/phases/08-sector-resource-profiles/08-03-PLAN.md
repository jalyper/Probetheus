**Test structure (approximately 10-12 tests):**

**PROF-01: Profile assignment on discovery**
1. "newly discovered sector receives a resourceProfile" - Create sector via `sectorManager.initializeSector(5, 5, true)`, verify it has `resourceProfile` with `type`, `spawnRateMultiplier`, and `assignedDistance` properties
2. "resourceProfile type is one of the five valid types" - Create multiple sectors, verify each profile type is one of: 'balanced', 'mineral-rich', 'data-rich', 'artifact-rich', 'probethium-rich'
3. "existing sector without profile gets one on discovery" - Manually create a sector without resourceProfile, call `discoverSector()`, verify profile is assigned

**PROF-02: Profile affects spawn rates**
4. "spawn rate multiplier is correct for each profile type" - Create sectors with known profiles, verify multipliers: balanced=1.0, mineral-rich=1.5, data-rich=1.5, artifact-rich=1.5, probethium-rich=0.8

**PROF-03: Distance-weighted distribution**
5. "close sectors are mostly balanced" - Generate 200 sectors at distance < 5 (e.g., coordinates 0-4), count profiles. Balanced should be >40% (with tolerance for RNG). Use 5-60% range for tolerance.
6. "far sectors have more specialized profiles" - Generate 200 sectors at distance > 10 (e.g., coordinates 8-12), count profiles. Balanced should be <50%. Specialized profiles should be more common than close sectors.

**PROF-04: Lucky early finds**
7. "probethium-rich sectors possible at any distance" - Generate 500 sectors at distance < 5. At least 1 should be probethium-rich (2% base rate * 500 = ~10 expected). This is a statistical test - use generous tolerance.

**PROF-05: Mining station output**
8. "mineral-rich sector station produces minerals" - Set up sector with `resourceProfile.type = 'mineral-rich'`, place station, call `getStationOutputResource()`, verify returns 'minerals'
9. "probethium-rich sector station produces probethium" - Same pattern, verify returns 'probethium'
10. "balanced sector station produces mixed" - Same pattern, verify returns 'mixed'

**PROF-06: Probethium exclusivity**
11. "only probethium-rich sectors produce probethium from mining" - Test all 5 profile types via `getStationOutputResource()`. Only 'probethium-rich' should return 'probethium'.

**Backward compatibility**
12. "sector without resourceProfile defaults to probethium output" - Create station in sector that has NO resourceProfile property. `getStationOutputResource()` should return 'probethium' (backward compatible fallback).

**Key patterns for test implementation:**
- Use `page.evaluate()` with direct game object access via `window.game`

- Use tolerance ranges: 5-60% for weighted probability checks (accounts for RNG variance)
- Access `window.game.sectorManager` for profile assignment
