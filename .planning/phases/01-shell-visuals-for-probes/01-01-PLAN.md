  <name>Task 1: Bridge shell visuals to probe.cosmetic</name>
  <files>src/ShellSystem.js</files>
  <action>
Add a `buildCosmeticFromShell(shell)` method to ShellSystem class that converts shell.visual properties to the probe.cosmetic format expected by rendering:

```javascript
buildCosmeticFromShell(shell) {
    const visual = shell?.visual || {};
    const defaultColor = '#00ffff';

    return {
        trailEnabled: true,
        trail: {
            length: 15,
            color: visual.trail || visual.color || defaultColor,
            width: visual.glow ? 4 : 3,
            opacity: visual.glow ? 0.95 : 0.9
        },
        bodyColor: visual.color || defaultColor,
        wingColor: visual.color || defaultColor,
        frontColor: visual.color || defaultColor,
        antennaColor: visual.color || defaultColor,
        glow: visual.glow || false,
        blinkSpeed: 1500
    };
}
```

Then modify `equipShellOnProbe(probe, shellId)` to apply cosmetic data:
- After setting `probe.shellId = shellId`, get the shell via `this.getShell('probes', shellId)`
- Call `buildCosmeticFromShell(shell)` and assign result to `probe.cosmetic`
- This ensures rendering immediately uses new visual properties

Also add a helper `refreshProbeCosmetic(probe)` that re-applies cosmetic from current shellId (useful for save/load scenarios).
