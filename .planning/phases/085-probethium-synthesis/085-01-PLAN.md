**Create src/SynthesisAnimationManager.js (NEW FILE):**

Create a class `SynthesisAnimationManager` that manages a queue of synthesis animations rendered on the galaxy canvas.

Constructor takes `eventBus`. Listens to `synthesis:triggered` to enqueue animations.

**Queue system:**
- `this.queue = []` -- array of pending animation objects `{ hub, startTime: null }`
- `this.currentAnimation = null` -- the active animation `{ hub, startTime, elapsed }`
- On `synthesis:triggered`: push to queue. If no current animation, start next.
- `startNextAnimation()`: shift from queue, set `startTime = Date.now()`, set as current.

**update(deltaTime) method:**
- If no current animation, return early
- Calculate `elapsed = Date.now() - this.currentAnimation.startTime`
- If elapsed >= 3000: animation complete, emit `synthesis:completed`, call `startNextAnimation()`
- Store elapsed on currentAnimation for render access

**render(ctx, viewOffset) method:**
- If no current animation, return early
- Get hub from currentAnimation, calculate screen position: `hub.x - viewOffset.x`, `hub.y - viewOffset.y`
- Call `ctx.save()` at start, `ctx.restore()` at end (CRITICAL -- prevents canvas state leaks)
- Calculate `progress = elapsed / 3000` (0.0 to 1.0)

**Phase 1: Charge-up (progress 0.0 to 0.33, elapsed 0-1000ms):**
- 12 purple/violet particles spiral inward toward hub center
- Each particle: `angle = (elapsed * 0.003) + (i * Math.PI * 2 / 12)`
- Spiral radius shrinks from 80px to 0 over the phase: `80 * (1 - phaseProgress)`
- Particle color: `hsla(270-290, 70%, 60%, opacity)` where opacity increases with progress
- Hub gets purple glow overlay: `rgba(148, 0, 211, 0.3 * pulseIntensity)` at radius 25-30px

**Phase 2: Flash (progress 0.33 to 0.5, elapsed 1000-1500ms):**
- White-to-gold flash circle expanding from hub center
- Flash starts white `rgba(255,255,255,alpha)`, transitions to gold `rgba(255,215,0,alpha)` over 500ms
- Flash radius expands from 30px to 80px
- Screen shake for first 300ms of this phase: 2-3px random offset applied via a `shakeOffset` property
  - `shakeOffset = { x: (Math.random()-0.5) * intensity, y: (Math.random()-0.5) * intensity }`
  - Intensity decays from 2.5 to 0 over 300ms: `2.5 * (1 - phaseElapsed / 300)`
  - IMPORTANT: Store shakeOffset as a property on the manager. Do NOT modify viewOffset directly. The game.js render() method should ADD shakeOffset to viewOffset TEMPORARILY when calling ctx.translate or when positioning entities.
  - After 300ms, reset shakeOffset to {x:0, y:0}
- Golden energy ring expanding outward: `strokeStyle = rgba(255,215,0, fading)`, radius grows from 0 to 100px

**Phase 3: Release (progress 0.5 to 1.0, elapsed 1500-3000ms):**
- Golden afterglow on hub fading out: `rgba(255,215,0, 0.4 * (1-phaseProgress))` at expanding radius 30-50px
- 16 golden particles drift outward from hub: from 30px to 90px distance
- Particles shrink and fade as they drift: size `2.5 * (1-phaseProgress)`, alpha `0.9 * (1-phaseProgress)`
- Each particle has slight wobble: `Math.sin(elapsed * 0.004 + i) * 8`

**Screen shake approach (safe, no viewOffset mutation):**
- SynthesisAnimationManager exposes `getShakeOffset()` returning `{x, y}` (0,0 when no shake)
- In game.js `render()`, BEFORE drawing entities, get shake offset and apply as canvas translate:
  ```javascript
  const shake = this.synthesisAnimation ? this.synthesisAnimation.getShakeOffset() : {x:0, y:0};
  if (shake.x !== 0 || shake.y !== 0) {
      this.ctx.save();
      this.ctx.translate(shake.x, shake.y);
  }
  // ... existing render code ...
  if (shake.x !== 0 || shake.y !== 0) {
      this.ctx.restore();
  }
  ```
  This avoids mutating viewOffset which would cause drift (Pitfall 5 from research).
