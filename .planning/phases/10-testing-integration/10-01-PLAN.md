  <action>
Create `tests/progression-gates.spec.js` with the following structure. Use `const { test, expect } = require('@playwright/test');` format (matching majority of existing tests).

Include the standard startGame helper (see research-gating.spec.js or exclusive-signals.spec.js for pattern):
- Click #newGameBtn, wait 1500ms, skip cutscene if visible, wait for #galaxyCanvas, wait 3000ms, dismiss tutorial panel.

Include a beforeEach that navigates to '/', clears localStorage, reloads, and waits for networkidle.

Write the following test groups with the gate test pattern (attempt before -> meet prerequisite -> verify after):

**1. Tutorial & Early Game Gates (5-6 tests):**

- `player starts with correct initial state (0 resources, 1 hub, 3 probes)` — Verify gameState.resources.minerals === 0, resources.data === 0, entities.reconHubs.length === 1, entities.probes.length === 3.

- `cannot build hub with insufficient minerals` — Set minerals to 99, attempt to build hub (call buildingSystem.canBuildHub() or check resource requirement). Verify it fails. Set minerals to 100, verify it succeeds.

- `cannot build mining station without 100 minerals and 50 data` — Set minerals to 100 but data to 49, verify station build blocked. Set data to 50, verify it succeeds.

- `cannot build shuttle without 50 minerals and 25 data` — Same pattern with shuttle resource requirements.

- `hub placement restricted to outbound path only` — Verify buildingSystem.isValidHubPlacement() rejects positions not on probe outbound path. Use programmatic check.

- `cannot install equipment before researching corresponding collector` — With no research, attempt to install mineral_collector on probe via EquipmentSystem. Verify it fails. Add auto_minerals to researched set, verify it succeeds.

**2. Research System Gates (5-6 tests):**

- `research system locked at game start` — Verify gameState.getResearchSystem().unlocked === false and researchAccessAllowed === false.

- `research requires BOTH points AND tutorial gate` — Give 5 research points but keep tutorial gate closed. Verify research stays locked. Then open tutorial gate, verify research unlocks.

- `cannot research child before parent` — Set up research system as unlocked with points. Attempt to research auto_minerals (child of auto_collectors). If auto_collectors isn't researched, it should fail. Research auto_collectors first, then auto_minerals succeeds.

- `research nodes require correct point costs` — Verify that researching a 1-point node deducts 1 point, a 3-point node deducts 3 points.

- `auto_all requires all 3 base collectors` — Research auto_minerals and auto_data but not auto_artifacts. Verify auto_all research fails. Research auto_artifacts, verify auto_all succeeds.

- `probethium_synthesis requires alien_tech parent` — Attempt probethium_synthesis without alien_tech researched. Verify it fails. Research alien_tech, verify probethium_synthesis succeeds.

**3. Equipment Gates (4 tests):**

- `mineral collector requires auto_minerals research` — Without research, verify EquipmentSystem.canEquip() or equivalent returns false for mineral_collector. With auto_minerals researched, returns true.

- `data collector requires auto_data, artifact collector requires auto_artifacts` — Same pattern for data_collector and artifact_collector.

- `universal collector requires auto_all research` — Without auto_all, verify universal_collector equip blocked.

- `cannot exceed 2 equipment slots per probe` — Equip 2 items on a probe (set up research first), attempt to equip 3rd. Verify it fails.

- `equipment installation requires correct resource cost (25 minerals, 50 for universal)` — Verify equipping with < 25 minerals fails, with >= 25 succeeds. Verify universal requires 50.

**4. Mining & Probethium Gates (3-4 tests):**

- `cannot collect probethium before mining station + shuttle` — Without any mining infrastructure, verify probethium stays at 0. (Programmatic check on mining system state.)

- `synthesis button hidden until probethium_synthesis researched` — Check that the synthesis button does not exist in DOM or is hidden. Research probethium_synthesis (and its parent alien_tech). Verify button appears.

- `synthesis requires 5+ exotic minerals` — With research done but exoticMinerals < 5, verify synthesis button disabled. Set exoticMinerals to 5, verify it's enabled.

**5. Remnant NPC Gates (2-3 tests):**

- `remnants dont spawn with fewer than 2 explored sectors` — Set up 1 explored sector. Call RemnantManager.checkSpawnConditions() (or equivalent). Verify it returns false. Add second explored sector and a mining station. Verify conditions now met.

- `remnants dont spawn with 0 mining stations` — Set up 2+ explored sectors but no mining stations. Verify spawn conditions fail. Add station, verify they pass.

- `specific NPC requirements enforced` — Verify Whisperer requires 3+ mining stations, Archivist requires 5+ explored sectors (check canSpawnSpecificNPC or equivalent method).

**6. Sector & Signal Gates (3-4 tests):**

- `standard sectors spawn no exclusive signals (cross-system gate)` — Create Standard sector, generate 200 signals. Verify 0 exclusive types.

- `non-Standard sectors spawn ONLY their specific exclusive type` — Create Resource-Rich sector, generate 500 signals. Verify only ore_vein appears as exclusive (no data_cache, relic, exotic_crystal).

- `discovery bonus signals only spawn on first discovery` — Call spawnDiscoveryBonusSignals for a sector. Clear signals. Call again for same sector. Verify no new exclusive signal spawned on second call.

- `sector discovery awards +1 research point` — Track research points before and after sector discovery. Verify +1 increment.

IMPORTANT IMPLEMENTATION NOTES:
- Use page.evaluate() for all game state checks (no UI clicking for gate tests).
- For building/equipment checks, call the actual game methods (buildingSystem, EquipmentSystem) rather than clicking UI buttons.
- Each test should be self-contained (beforeEach clears state).
- Follow the gate pattern: verify blocked state, meet prerequisite, verify unblocked.
- Reference 10-CONTEXT.md resource costs table for exact values.
- Research system requires BOTH `research.unlocked = true` AND `tutorialManager.researchAccessAllowed = true` (Pitfall 3 from RESEARCH.md).
- Equipment is array format on probes: `probe.equipment = []` (Pitfall 4).
- Always include resourceProfile on programmatic sectors (Pitfall 8).

Target: 20-25 tests. Prioritize breadth of gate coverage over depth of individual gates.
