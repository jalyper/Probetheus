**TEST-01: Probe bonus tests** (one test per bonus type, 8 total):

Each test follows this pattern:
1. Start game
2. Use `page.evaluate()` to get a probe, own a shell, equip it
3. Verify the bonus is resolved correctly via `getEntityBonus()`
4. Verify the game mechanic is affected (where possible to test without full gameplay)

Test cases:
- **dataSignalDiscovery**: Equip calculator_core on probe, verify `getEntityBonus('probes', probe, 'dataSignalDiscovery')` returns 10 (uncommon rarity)
- **signalRange**: Equip echo_receiver on probe, verify `getEntityBonus('probes', probe, 'signalRange')` returns 10
- **rareSignalChance**: Equip echo_receiver, verify `getEntityBonus('probes', probe, 'rareSignalChance')` returns 5 (secondary bonus)
- **probeDurability**: Equip reinforced_hull, verify probe.maxDamage is higher than base (base 3, with 10% bonus should be Math.floor(3 * 1.10) = 3... so test with a higher rarity: equip fortress_probe (epic, 20%), verify Math.floor(3 * 1.20) = 3... For better testability: set probe.baseMaxDamage to 10 first, then equip, verify maxDamage = Math.floor(10 * 1.20) = 12)
- **asteroidSurvival**: Equip reinforced_hull, verify `getEntityBonus('probes', probe, 'asteroidSurvival')` returns 5
- **artifactDiscovery**: Equip relic_seeker, verify `getEntityBonus('probes', probe, 'artifactDiscovery')` returns 10
- **explorationRewards**: Equip relic_seeker, verify `getEntityBonus('probes', probe, 'explorationRewards')` returns 5
- **exoticYield**: Equip void_touched, verify `getEntityBonus('probes', probe, 'exoticYield')` returns 10

For at least 2 bonus types, also verify the GAMEPLAY EFFECT:
- signalRange: After equipping, check that the collection range calculation in context yields a higher value. E.g., `80 * (1 + 10/100) = 88`
- probeDurability: After equipping (with manipulated baseMaxDamage), check probe.maxDamage is correct
