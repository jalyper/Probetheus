**HBON-01: researchSpeed as cost reduction** (src/ResearchManager.js, around line 375)

Read the full ResearchManager.js file to find the research completion/purchase logic. The research is instant (spend points, complete node). The integration point is where `research.points -= node.cost` happens.

Implement cost reduction:
```javascript
const researchShellBonus = window.game?.shellSystem ? window.game.shellSystem.getEntityBonus('hubs', null, 'researchSpeed') : 0;
const effectiveCost = Math.max(1, Math.ceil(node.cost * (1 - researchShellBonus / 100)));
research.points -= effectiveCost;
```

Key rules:
- Minimum cost of 1 (never free research)
- Use `Math.ceil()` to round up (conservative for the player's benefit, rounding down would be MORE generous -- actually use Math.ceil to round UP the cost, which is less generous. Per RESEARCH.md recommendation: `Math.ceil(node.cost * (1 - bonus/100))`)
- Hub shells use category-level: `cosmetics.equippedShells.hubs`
- The `entity` parameter is null (hubs use category-level, same as stations)

Also check if there's a cost display anywhere that should show the reduced cost. If there is UI that shows "Cost: X", update it to show the reduced cost. But do NOT add bonus indicator UI in this plan (that's Phase 4 scope).

Read the file carefully -- there may be multiple places where cost is checked (can-afford check) and spent (actual deduction). Both should use the same reduced cost formula so the player can afford things at the reduced price.
